# NeuroFix — AI Photo Restoration Bot

Telegram-бот для восстановления, улучшения и анализа фотографий. Пользователь отправляет снимок — система автоматически анализирует его и запускает нужную цепочку обработки на GPU.

## Что умеет

| | Функция | Описание |
|-|---------|----------|
| 🔄 | **Реставрация** | Устраняет царапины, трещины, выцветание на старых фото |
| ⚡ | **Улучшение качества** | Апскейл ×4 с восстановлением деталей лиц и текстур |

## Архитектура

Система построена на микросервисной архитектуре с асинхронной очередью задач. Каждый сервис изолирован в Docker-контейнере с доступом к GPU и общается с остальными только через Redis Streams — прямых зависимостей между контейнерами нет.

```
Пользователь
    │
    ▼
Telegram Bot (aiogram 3)
    │
    ├──► classify:tasks ──► Классификатор (OpenCLIP ViT-B/32)
    │         └── classify:result:{id} ──► Bot
    │
    ├──► photo:tasks ────► Worker    (реставрация)
    ├──► enhance:tasks ──► Enhancer  (улучшение качества)
    ├──► colorize:tasks ─► Colorizer (раскраска)
    └──► emotion:tasks ──► Emotion   (анализ портрета)
                │
                └── photo:results ──► Bot ──► Пользователь
```

## Пайплайн обработки

### Классификатор

При получении фото запускается собственный сервис классификации на базе CLIP ViT-B/32. Он выполняет три последовательные проверки:

**Детекция лица** — CLIP сравнивает эмбеддинг изображения с текстовыми промптами. Если лицо найдено — дальнейшие эвристики пропускаются. Без этого шага портреты ошибочно определялись как документы.

**Определение цветности** — анализ медианной насыщенности пикселей в пространстве HSV. Точнее CLIP-классификации для этой задачи.

**Детекция повреждений** — zero-shot через CLIP: промпты `"damaged scratched photo"` vs `"clean photo"`. Возвращает вероятность повреждения.

Результат сохраняется в Redis Hash с TTL — бот читает его без ожидания в очереди.

### Авто-пайплайн

На основе классификации система строит индивидуальную цепочку:

| Ч/Б | Повреждено | Цепочка обработки |
|:---:|:---:|---|
| ✓ | ✓ | реставрация → улучшение → раскраска |
| ✓ | ✗ | улучшение → раскраска |
| ✗ | ✓ | реставрация → улучшение |
| ✗ | ✗ | улучшение |

Состояние цепочки отслеживается в памяти бота по `task_id`. Когда приходит результат — автоматически запускается следующий шаг с новым `task_id`. Пользователь получает промежуточные результаты на каждом этапе.

### Маршрутизация задач

Все воркеры пишут результаты в единый стрим `photo:results`. Бот читает его и сопоставляет результаты с пользователями по `task_id` — без прямых HTTP-зависимостей между сервисами. Это позволяет обрабатывать задачи нескольких пользователей одновременно.

Фотографии передаются внутри стримов как base64. Все очереди ограничены по размеру (`MAXLEN=50`) для контроля памяти Redis.

### Анализ портрета

Два фреймворка работают совместно: InsightFace выравнивает лицо через `norm_crop` по 5 ключевым точкам, DeepFace анализирует эмоции на уже выровненном кропе. Каждый делает то, в чём сильнее.

## Быстрый старт

### 1. Клонировать

```bash
git clone https://github.com/WorDem125/neurofix-bot.git
cd neurofix-bot
```

### 2. Указать токен

```bash
cp .env.example .env
```

Вставить токен от [@BotFather](https://t.me/BotFather):

```
BOT_TOKEN=your_telegram_bot_token_here
```

### 3. Запустить

```bash
make start
```

Автоматически скачает модели и поднимет все контейнеры. Первый запуск — 30–60 минут.

## Требования

- Docker + Docker Compose
- NVIDIA GPU с поддержкой CUDA

```bash
make start           # Запустить
make stop            # Остановить
make restart         # Перезапустить
make status          # Статус контейнеров
make logs            # Логи всех сервисов
make logs-bot        # Логи конкретного сервиса
make logs-worker     # (worker / enhancer / colorizer / classifier / emotion)
make clean           # Удалить контейнеры и volumes
```
